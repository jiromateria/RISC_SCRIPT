# Решение для кодирования инструкций

На входе скрипт получает JSON-файл (формат описан в задании). На основе этого файла частично заполняются переменные и массивы определённых классов.

## Основные классы

- **Field** – поле в команде (R1, R2, OPCODE, F и т.д.). Хранит `lsb`, а также `degree` – связность узла в графе конфликтов (число других полей, с которыми данное поле может встречаться в одной команде).
- **Command** – конкретная команда, используется для нумерации инструкций в едином формате.
- **Format** – хранит информацию о каждом формате.

## Алгоритм

Задача в общем виде является NP-сложной, поэтому используются эвристики и предположение, что конечные инструкции должны быть RISC-формата.

1. После обработки всех данных строится **матрица конфликтов** (или граф конфликтов, но матрица проще). Элемент `(i, j)` равен 1, если поля `i` и `j` встречаются в одном формате (их позиции не могут пересекаться).
2. Поля сортируются в порядке:
   - Фиксированное ли значение;
   - Связность (число конфликтов);
   - Длина;
   - Имя.
   
   При этом поле **OPCODE** всегда имеет приоритет (в соответствии с RISC-архитектурой).
3. Поля с фиксированным значением располагаются слева направо, с нефиксированным – справа налево. Выполняются проверки, чтобы конфликтующие поля не пересекались.
4. После расстановки всех полей выполняется расширение полей или добавление `RES0`, `RES1` и т.д., где это возможно.
5. Результат выводится в требуемом формате.
